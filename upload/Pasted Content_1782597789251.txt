# 5. ARCHITEKTÚRA SIMULÁCIE

Simulácia musí byť:

* deterministická,
* serializovateľná,
* testovateľná,
* oddelená od renderovania,
* pripravená na autoritatívny server,
* založená na fixed timestep.

Použi:

```ts
export const SIMULATION_HZ = 60;
export const FIXED_DT = 1 / SIMULATION_HZ;
```

Hlavný loop:

```ts
accumulator += frameDelta;

while (accumulator >= FIXED_DT) {
  const inputFrame = inputBuffer.consumeForTick(simulation.tick);
  simulation.step(inputFrame, FIXED_DT);
  accumulator -= FIXED_DT;
}

renderer.render(
  simulation.previousState,
  simulation.currentState,
  accumulator / FIXED_DT
);
```

Simulácia používa celé tick čísla.

Gameplay timeouty neukladaj v náhodných milisekundách.

Použi:

```ts
startedTick
contactTick
expiresTick
recoveryUntilTick
```

# 7. SINGLE SOURCE OF TRUTH PRE LOPTU

Jediný zdroj vlastníctva lopty:

```ts
ball.ownerId
```

Nevytváraj paralelný stav:

```ts
player.hasBall
```

Stav lopty:

```ts
type BallMode =
  | 'CONTROLLED'
  | 'FREE'
  | 'PASS'
  | 'SHOT'
  | 'AERIAL'
  | 'GOALKEEPER_HELD'
  | 'RESTART'
  | 'OUT_OF_PLAY';

interface BallState {
  mode: BallMode;
  ownerId: number | null;
  previousOwnerId: number | null;
  lastTouchPlayerId: number | null;
  lastTouchTeamId: number | null;

  position: Vec3;
  previousPosition: Vec3;
  velocity: Vec3;
  spin: Vec3;

  ownerSinceTick: number;
  releasedAtTick: number;
  aftertouch: AftertouchState | null;
}
```

Invarianty:

* `CONTROLLED` musí mať ownerId,
* `GOALKEEPER_HELD` musí mať ownerId,
* `FREE`, `PASS`, `SHOT` a `AERIAL` nesmú mať ownerId,
* jeden hráč nemôže vlastniť loptu súčasne s iným,
* počas wind-up zostáva vlastníctvo hráčovi,
* pri kontakte kopu sa vlastníctvo uvoľní presne raz.

Pridaj:

```ts
assertBallState(state);
```

# 8. POHYB HRÁČOV

Pohyb musí byť arkádový, okamžitý a čitateľný.

Každý hráč má:

```ts
interface PlayerMotionState {
  position: Vec2;
  previousPosition: Vec2;
  velocity: Vec2;
  facing: number;
  desiredDirection: Vec2;
  movementMode:
    | 'IDLE'
    | 'WALK'
    | 'RUN'
    | 'SPRINT'
    | 'BRAKE'
    | 'ACTION'
    | 'RECOVERY';
}
```

Podporuj:

* analógovú intenzitu,
* digitálny keyboard preset,
* akceleráciu,
* brzdenie,
* rýchle otočenie,
* mierny postih za 180° obrat,
* samostatnú rýchlosť s loptou,
* sprint bez lepkavého zastavenia.

Rozsahy analógového vstupu:

```text
0.00–0.18 = státie
0.18–0.45 = chôdza
0.45–0.80 = beh
0.80–1.00 = rýchly beh
sprint + vysoká intenzita = šprint
```

Neaplikuj trest za otočenie opakovane každý tick.

# 40. TESTY

## Determinizmus

* rovnaký seed a vstupy vytvoria rovnaký stav,
* simulácia je rovnaká pri rôznych render FPS,
* serializácia a deserializácia zachová výsledok.

## Input

* krátky tap sa nestratí,
* edge event sa spracuje raz,
* catch-up ticks nezopakujú akciu.

## Lopta

* ownership invarianty vždy platia,
* kontrolovaná lopta neodskakuje náhodne,
* kop uvoľní loptu presne raz,
* predikcia zodpovedá reálnej trajektórii.

## Prihrávky

* krátka prihrávka trafí otvoreného príjemcu,
* blokovaný koridor sa vyhodnotí správne,
* príjemca môže loptu spracovať,
* prudká prihrávka môže spôsobiť heavy touch.

## Player selection

* majiteľ lopty je ovládaný,
* manuálny lock sa rešpektuje,
* switch neosciluje,
* voľná lopta používa interception estimate.

## AI

* iba jeden primary presser,
* obrancovia držia tvar,
* spoluhráči vytvárajú možnosti,
* AI dokáže prihrať a vystreliť,
* AI nebehá všetkými hráčmi za loptou.

## Brankár

* zákrok závisí od trajektórie,
* catch a parry sú odlišné,
* dive môže zasiahnuť loptu,
* lopta mimo save zone nie je chytená.

## Bránka

* žrď odráža,
* brvno odráža,
* sieť tlmí,
* gól sa započíta iba pri správnom prechode čiary.
