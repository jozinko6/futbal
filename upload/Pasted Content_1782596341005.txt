# ARCADE FOOTBALL REBUILD

## Clean-room analýza YSoccer + prestavba projektu `jozinko6/futbal`

Pracuješ na existujúcom repozitári:

`jozinko6/futbal`

Ako technickú referenciu analyzuj dekompilovaný projekt:

`boltvious/ysoccer`

Referenčný projekt je dekompilovaný Java/libGDX program pôvodne získaný z `ysoccer.jar`.

Tvojou úlohou nie je mechanicky preniesť Java kód.

Tvojou úlohou je:

1. pochopiť správanie jednotlivých systémov YSoccer,
2. vytvoriť ich vlastnú čistú TypeScript implementáciu,
3. opraviť súčasnú nehrateľnú simuláciu projektu `futbal`,
4. vytvoriť rýchly a responzívny arkádový futbal,
5. použiť YSoccer a Sensible Soccer ako gameplayovú referenciu,
6. použiť Taito Cup Finals ako hlavnú vizuálnu a prezentačnú referenciu,
7. zachovať originálny názov, grafiku, tímy, hudbu a assety.

Výsledok musí byť originálny duchovný nástupca klasických arkádových futbalových hier.

# 5. FIXED-STEP INPUT QUEUE

Súčasné tlačidlové edge udalosti sa nesmú spotrebovať v render frame, počas ktorého neprebehne simulačný tick.

Rozdeľ vstup na:

```ts
interface ContinuousInput {
  moveX: number;
  moveY: number;
  aimX: number;
  aimY: number;
  sprintHeld: boolean;
  actionHeld: boolean;
  modifierHeld: boolean;
}

interface InputEdges {
  actionPressed: boolean;
  actionReleased: boolean;
  tacklePressed: boolean;
  switchPressed: boolean;
  specialPressed: boolean;
  pausePressed: boolean;
}
```

InputManager musí uchovávať edge udalosti vo fronte až do najbližšieho simulačného ticku.

Použi:

```ts
while (accumulator >= FIXED_DT) {
  const input = inputManager.consumeForSimulationTick();
  step(state, input, FIXED_DT);
  accumulator -= FIXED_DT;
}
```

Pravidlá:

* continuous input dostane každý tick,
* edge input dostane iba prvý príslušný tick,
* krátke stlačenie sa nesmie stratiť,
* catch-up ticks nesmú akciu zopakovať,
* keyboard, gamepad aj touch používajú rovnakú abstrakciu.

Pridaj testy pri renderovacích frekvenciách:

* 30 FPS,
* 60 FPS,
* 120 FPS,
* 144 FPS,
* 240 FPS.

Výsledok simulácie musí byť konzistentný.

# 6. JEDINÝ ZDROJ VLASTNÍCTVA LOPTY

Odstráň konflikt medzi:

```ts
ball.ownerId
player.hasBall
```

Jediným autoritatívnym zdrojom je:

```ts
ball.ownerId
```

Použi helper:

```ts
function playerHasBall(state: MatchState, playerId: number): boolean {
  return state.ball.ownerId === playerId;
}
```

`player.hasBall`:

* úplne odstráň,
* alebo ho ponechaj iba ako odvodené read-only pole pre kompatibilitu,
* nikdy ho samostatne nezapisuj,
* gameplay logika ho nesmie čítať.

Vytvor explicitné režimy lopty:

```ts
type BallMode =
  | 'CONTROLLED'
  | 'FREE'
  | 'PASS'
  | 'SHOT'
  | 'AERIAL'
  | 'GK_HELD'
  | 'RESTART'
  | 'OUT_OF_PLAY';
```

Rozšír stav:

```ts
interface BallState {
  mode: BallMode;
  ownerId: number | null;
  previousOwnerId: number | null;
  lastTouchPlayerId: number | null;
  lastTouchTeam: Team | null;
  ownerSinceTick: number;
  releasedAtTick: number;
}
```

Invarianty:

* `CONTROLLED` musí mať ownerId,
* `GK_HELD` musí mať ownerId brankára,
* `FREE`, `PASS`, `SHOT` a `AERIAL` nesmú mať ownerId,
* akcia sa môže začať iba skutočnému majiteľovi,
* vlastníctvo sa uvoľní presne v kontaktnom ticku,
* počas wind-up vlastníctvo zostáva hráčovi,
* proximity systém nesmie odobrať kontrolovanú loptu.

V development režime pridaj:

```ts
assertBallInvariants(state);
```

Pri chybe vypíš:

* tick,
* mode,
* ownerId,
* previousOwnerId,
* activePlayerId,
* player action,
* action phase,
* pozíciu a rýchlosť lopty.

Predikciu používaj pre:

* výber hráča,

# 11. POHYB HRÁČA

Analógová intenzita musí ovplyvňovať rýchlosť.

Použi:

```ts
const magnitude = clamp(length(moveX, moveY), 0, 1);
```

Rozsahy:

```text
0.00–0.20 = stoj
0.20–0.45 = chôdza
0.45–0.78 = klus
0.78–1.00 = beh
plné vychýlenie + sprint = šprint
```

Klávesnica môže používať digitálny arkádový profil.

Pridaj:

* deadzone,
* response curve,
* samostatnú rýchlosť s loptou,
* rýchle, ale nie okamžité otočenie,
* mierne spomalenie pri otočení o 180°.

Odstráň opakované násobenie velocity pri prudkom otočení.

Namiesto toho:

* znižuj dostupné zrýchlenie podľa uhla,
* aplikuj jednorazový braking impulse,
* sleduj začiatok sharp-turn stavu,
* neaplikuj trest každý tick.

Hráč musí pôsobiť arkádovo a okamžite reagovať.

# 12. KONTEXTOVÉ OVLÁDANIE

Vytvor predvolený režim:

```text
CLASSIC ARCADE
```

Použi tri hlavné tlačidlá.

## ACTION

S loptou:

* krátky tap mimo streleckej pozície = krátka prihrávka,
* krátky tap v dobrej streleckej pozícii a smerom na bránu = rýchla strela,
* podržanie = silná prihrávka alebo silná strela,
* pri vysokej lopte = hlavička alebo volej,
* modifier + action = lob alebo chip.

Bez lopty:

* pri nízkej lopte = standing tackle alebo poke,
* pri vysokej lopte = hlavička,
* pri lopte vo vhodnej vzdialenosti = blok,
* vedľa súpera = shoulder challenge.

## TACKLE

Bez lopty:

* slide tackle.

S loptou:

* sprint alebo skill modifier podľa nastavenia.

## SWITCH/SPECIAL

Počas hry:

* manuálne prepnutie.

Počas prerušenia:

* otvorenie Special Menu.

Pri dostupnom kapitánskom špeciále:

* aktivácia Super Shot.

V debug overlayi zobraz:

```text
Context action: SHORT_PASS
Reason: teammate in input cone, outside shooting zone
Alternatives: DRIVEN_PASS, HOLD
```

Akcia nesmie byť náhodne premenená na inú.

# 16. STABILNÉ PREPÍNANIE HRÁČA

Vytvor:

```text
src/game/simulation/playerSelection.ts
```

Ak tím drží loptu:

* ovládanie sa prepne na majiteľa,
* pri plánovanej prihrávke možno označiť predpokladaného príjemcu,
* skutočné ovládanie sa odovzdá v jasne definovanom okamihu.

Ak súper drží loptu:

* vyber najlepšieho obrancu medzi loptou a vlastnou bránou,
* zohľadni predikovaný čas k lopte,
* zohľadni pressing assignment.

Ak je lopta voľná:

* vyber hráča s najlepším interception estimate,
* nie automaticky najbližšieho podľa XY.

Pridaj:

```ts
lastSwitchTick
switchCooldownTicks
manualLockUntilTick
candidateId
candidateScore
currentScore
```

Použi hysteréziu:

```ts
candidateScore >= currentScore + SWITCH_MARGIN
```

Manuálne prepnutie:

* má okamžitú prioritu,
* zostane zamknuté približne 0,7–0,9 sekundy,
* nesmie byť hneď prepísané automatikou.

Automatické prepnutie:

* maximálne raz za cooldown,
* nesmie oscilovať medzi dvomi hráčmi,
* neprepína na brankára bez vážneho dôvodu.

# 23. KAMERA

# 30. PRÍSNE TESTY

## Input

* krátky tap sa nestratí pri 144 FPS,
* edge sa spracuje presne raz,
* catch-up tick nezopakuje prihrávku.

## Ownership

* `CONTROLLED` vždy má ownerId,
* `FREE` nikdy nemá ownerId,
* po kontakte strely je ownerId null,
* hráč bez vlastníctva nemôže začať kop,
* žiadny desync po 100 000 tickoch.

## Dribling

* normálny beh nespôsobí stratu,
* šprint nespôsobí náhodnú stratu,
* vizuálna poloha lopty nemení vlastníctvo,
* tackle jasne ukončí kontrolu.

## Trajectory

* neutral trajectory zodpovedá skutočnej fyzike,
* left/right spin zakriví loptu správne,
* predictor nemení aktívny state,
* cache sa invaliduje po kontakte.

## Interception

* rýchly hráč dosiahne bod skôr,
* hráč bežiaci opačne dostane turn penalty,
* vysokú loptu nemožno spracovať nohou,
* brankárska zóna sa započíta.

## Player selection

* majiteľ lopty sa stane aktívnym,
* manuálny lock sa rešpektuje,
* switch neosciluje,
* kandidát musí prekonať hysteréziu.

## Goalkeeper

* catch iba v catch zone,
* parry ponechá loptu živú,
* dive môže fyzicky zasiahnuť loptu,
* rush mení velocity smerom k interceptu,
* lopta mimo dosahu nie je chytená.

## AI

* iba jeden primary presser,
* cover assignment sa neprepíše,
* spoluhráči vytvárajú prihrávkové možnosti,
* AI neprihráva cez očividne blokovanú dráhu,
* formácia reaguje na zónu lopty.

## Prezentácia

* krátka prihrávka nespustí Power Shot shake,
* shake sa aplikuje iba raz,
* replay buffer neprekročí limit.

Nepoužívaj neurčité testy typu:

```ts
expect(controlled || close).toBe(true);
```

Kontroluj presný požadovaný výsledok.

# 33. DEFINITION OF DONE

Projekt je dokončený až keď:

1. Krátke vstupy sa nestrácajú.

2. Lopta má iba jeden autoritatívny zdroj vlastníctva.

3. Dribling je spoľahlivý.

4. Hráč sa ovláda okamžite a predvídateľne.

5. Analógová intenzita mení rýchlosť.

6. Prudké otočenie nespôsobuje lepkavé zastavenie.

7. Prihrávky majú bezpečné a čitateľné targeting pravidlá.

8. Bežná prihrávka sa spoľahlivo spracuje.

9. Aftertouch je použiteľný, ale nie prehnaný.

10. Predikcia lopty zodpovedá skutočnej fyzike.

11. Auto-switch používa predikovaný čas k lopte.

12. Manuálne prepnutie zostáva stabilné.

13. Tímy zachovávajú formáciu.

14. Iba jeden obranca je primary presser.

15. Spoluhráči vytvárajú možnosti na prihrávku.

16. Brankár fyzicky zasahuje loptu save zónami.

17. Žrde, brvno a sieť majú odlišné kolízie.

18. Jedno kontextové tlačidlo je dostatočné na základnú hru.

19. Hra je zábavná s vypnutými efektmi.

20. 7v7 zápas funguje pred rozšírením na 11v11.
